# 발표 구성안

## 1. Problem

기존 No-code AI 에이전트 도구는 사용은 쉽지만, 특정 작업에서의
제어성, 일관성, 확장성이 부족합니다.

- Prompt 중심 방식은 같은 질문에도 다른 결과가 나올 수 있습니다.
- 외부 API 호출을 항상 보장하기 어렵습니다.
- 같은 유형의 작업을 반복할 때 매번 prompt나 설정을 조정해야 합니다.

## 2. Baseline

전통적인 Python 구현에서는 다음 과정을 직접 작성해야 합니다.

1. 사용자 입력 받기
2. 도시 추출
3. 날짜 판단
4. 도시 좌표 조회
5. 날씨 API 호출
6. API 응답 파싱
7. 온도와 강수확률 기반 옷 추천
8. 최종 문장 생성

예상 수치:

| 항목 | 수치 |
|---|---:|
| 구현 단계 | 8-12 |
| 코드 라인 | 약 100-150줄 |
| 초보자 구현 시간 | 30-60분 |

## 3. Limitation

범용 No-code 도구는 코드를 줄여주지만, 특정 작업에서는 한계가 있습니다.

| 문제 | 설명 |
|---|---|
| Prompt 불안정성 | 같은 입력에도 답변 형식과 내용이 달라질 수 있음 |
| 도구 호출 불확실성 | 항상 외부 API를 호출한다고 보장하기 어려움 |
| 흐름 제어 어려움 | Plan/Act 순서를 고정하기 어려움 |
| 낮은 재사용성 | 비슷한 작업마다 다시 설정해야 함 |

## 4. Proposed Method

본 프로젝트는 템플릿 기반 Agent 생성 방식을 제안합니다.

> 템플릿 기반으로 고정된 실행 흐름과 도구 호출을 보장하는 AI 에이전트 생성 시스템

템플릿 구성:

| 단계 | 역할 |
|---|---|
| Plan | 사용자 입력에서 도시, 날짜, 작업 유형 추출 |
| Act | 템플릿에 바인딩된 Weather API 호출 |
| Compose | 규칙 기반으로 날씨 요약과 옷 추천 생성 |

## 5. Demo

입력:

```text
서울 내일 날씨 알려주고 옷 추천해줘
```

실행 흐름:

```text
Plan: 도시=서울, 날짜=내일, 작업=날씨 조회 + 옷 추천
Act: Weather API 호출
Compose: 날씨 요약 + 옷 추천 생성
```

강조 문장:

> 이것은 단순한 챗봇 응답이 아니라, 정해진 흐름에 따라 외부 도구를 호출하고 결과를 생성하는 작업 실행형 Agent입니다.

## 6. Comparison

| 항목 | 전통 구현 | 범용 No-code 도구 | 본 프로젝트 |
|---|---|---|---|
| 코드 작성 | 필요 | 불필요 | 불필요에 가까움 |
| 도구 호출 제어성 | 높음 | 낮음, prompt 의존 | 높음, 템플릿 바인딩 |
| 흐름 안정성 | 높음 | 낮음 | 높음 |
| 구축 시간 | 김 | 짧음 | 가장 짧음, 템플릿 재사용 |
| 재사용성 | 낮음 | 중간 | 높음 |

## 7. Conclusion

본 시스템은 단순 챗봇이 아니라, 외부 도구 호출과 고정된 실행 흐름을 통해
일관된 작업 수행이 가능한 AI Agent를 효율적으로 생성할 수 있습니다.

## 예상 질문

### Q1. GPT Builder와 무엇이 다른가요?

GPT Builder는 주로 prompt에 의존하기 때문에 도구 호출과 실행 흐름이 항상
안정적이지 않습니다. 저희는 템플릿을 통해 실행 흐름과 도구 호출을 고정하여
일관성을 높였습니다.

### Q2. 기존 No-code 플랫폼을 사용하지 않은 이유는 무엇인가요?

기존 플랫폼은 범용성은 높지만 특정 작업에 최적화된 제어성과 재사용성에는
한계가 있습니다. 저희는 특정 작업 유형에 맞는 재사용 가능한 템플릿을 제공하는
방식에 초점을 맞췄습니다.

### Q3. 단순한 날씨 Demo 아닌가요?

날씨 Demo는 MVP입니다. 핵심은 날씨 기능 자체가 아니라 템플릿 기반 실행 흐름과
도구 호출 제어 가능성을 검증하는 것입니다.
