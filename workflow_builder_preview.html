Personalized Outfit Workflow Builder Preview

This static page shows how the current Python sample workflow maps to a visual Flowise/Dify/Langflow-style builder. The runnable local prototype is run_builder_app.cmd.
1. Parser

Request Parser Node

Extracts city, date, user id, purpose, and style from the natural language request.

2. Question

Question Node

Checks missing context. If city, date, or purpose/style is missing, it asks a follow-up question first.

3. Tool

Weather Tool Node

Calls the weather API and normalizes temperature, condition, and precipitation information.

6. Output

Compose Node

Renders the final answer with weather, shopping preference, and ranked outfit recommendations.

5. Decide

Recommendation Node

Combines weather and shopping analysis, then ranks owned items and additional suggestions.

4. Data

Shopping History Analysis Node

Loads simulated purchase records and extracts preferred styles, colors, categories, and items.

Project Position

What this is

A running sample workflow for personalized outfit recommendation. It proves the execution structure before building a full visual Builder.

What this is not

It is not yet a production Builder. The current semester scope is sample workflow implementation, local template-based Builder Prototype, tool comparison, and builder mapping.

Flowise PoC Mapping

  • configs/outfit_workflow.json: current executable workflow
  • configs/builder_templates/outfit_recommendation_template.json: builder template
  • configs/builder_templates/commute_outfit_template.json: second template
  • run_builder_app.cmd: runnable local Builder Prototype
  • configs/flowise_poc_mapping.json: builder-node mapping
  • deliverables/builder_tool_comparison_kr.md: tool comparison

Core Answer

Existing builders are general-purpose. Our contribution is the recommendation-domain workflow template: question, weather, shopping analysis, ranking, and composed output.
