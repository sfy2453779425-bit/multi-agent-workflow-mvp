<Summary in English> 氠瑢

Project Goal and Objectives

 The goal of this project is to implement a personalized outfit recommendation MVP for travel and daily scenarios. The system accepts a natural-language request, identifies missing context, retrieves weather data, analyzes shopping history, and generates ranked outfit recommendations through a JSON-defined 6-Node Workflow.

Project Contents 

 The project consists of configuration-driven workflow execution, request parsing, clarification question generation, Open-Meteo weather integration, local shopping-history analysis, temperature-rule matching, recommendation ranking, desktop UI, web UI, and workflow trace visualization. The default pipeline is Request Parser, Question, Weather Tool, Shopping History Analysis, Recommendation, and Compose.

Anticipated Results and Impacts on Industry

 The result is a working MVP that proves how reusable workflow nodes can combine external context and user-owned item data. It can be extended to travel platforms, fashion commerce services, and personal shopping assistants by connecting real shopping APIs, larger recommendation rules, and a visual workflow builder.

Purpose

(500자내외)

 This project develops a personalized outfit recommendation Workflow MVP for travel and daily use. The system receives a natural-language request, checks whether city, date, purpose, or style information is missing, asks follow-up questions when needed, and then combines weather data and shopping history to generate ranked outfit recommendations. The main objective is to prove that a user request can be handled through six small JSON-defined Workflow Nodes rather than one monolithic recommendation function

Contents

(1000자내외)

This project consists of the following sub tasks.

【1st task】Design and implementation of a JSON-based 6-Node Workflow
(1) Define outfit_workflow.json to describe Request Parser, Question, Weather Tool, Shopping History Analysis, Recommendation, and Compose Nodes.
(2) Implement MultiAgentWorkflowEngine to execute the configured Nodes sequentially and keep execution context and trace data.

【2nd task】Development of context-aware outfit recommendation logic
(1) Parse natural-language input to extract city, date, user, and intent keywords.
(2) Call Open-Meteo Geocoding and Forecast APIs to normalize weather condition, temperature, and precipitation probability.
(3) Analyze local shopping history to identify preferred styles, colors, categories, and owned items.
(4) Apply temperature rules and conditional extras such as umbrella or thin outerwear to produce ranked recommendations.

【3rd task】Implementation of demo interfaces and validation
(1) Provide a desktop demo that runs without browser or localhost.
(2) Provide a web demo that shows user input, workflow output, 6-Node trace, loaded config, and shopping-history analysis.
(3) Prepare tests for single-Config and Workflow behavior, including clarification short-circuit cases.

입력 예시 ‘옷 추천해줘’는 도시 정보가 부족하므로 추가 질문을 먼저 반환한다.
